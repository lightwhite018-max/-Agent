# 劳动者港湾智能助手

基于 PRD V1.0 的 V0.5 可交互原型。当前目标是跑通“自然语言/快捷需求 -> 港湾推荐 -> 详情 -> 文字反馈 -> 工单生成 -> 简易管理”的 MVP 核心链路。

## 本地运行

```bash
pnpm install
pnpm dev
```

Windows PowerShell 如果提示找不到 `pnpm`，可以直接运行：

```powershell
.\start-dev.ps1
```

## 当前能力

- 规则 Agent：识别喝水、充电、休息、避雨、如厕、反馈等需求。
- 样例港湾：内置 HB001-HB004 验收数据。
- 手动位置：模拟拒绝定位后的区县/商圈选择。
- 推荐规则：按开放状态、设施匹配、设施状态、步行时间和拥挤程度排序。
- 降级推荐：无完全匹配结果时展示开放但不完全匹配的备选方案。
- 详情展示：开放状态、设施状态、更新时间、导航降级提示。
- 反馈闭环：文字反馈生成反馈编号和工单编号。
- 日志预览：展示推荐结果数、是否降级和位置来源。
- 简易管理入口：查看港湾、修改港湾开放状态、修改设施状态、查看工单。
- 本地持久化：港湾状态和工单保存到当前浏览器，刷新后保留，并支持重置样例数据。
- Seed 数据层：港湾表和设施表分离，运行时组装成页面数据，方便后续迁移数据库/API。
- 服务接口层：页面通过 `prototypeApi` 调用推荐、反馈、管理和持久化能力，方便后续替换真实后端。
- 组件化页面：需求识别、推荐结果、详情、反馈、管理入口已拆成独立组件。
- 视图分区：用户端体验和管理端入口拆成两个标签页，便于演示核心链路。
- 验收面板：内置 P0-M 验收清单，展示已覆盖、演示增强和待接入能力。
- 演示状态栏：展示推荐结果、当前港湾、待处理工单和异常设施，便于演示时快速说明原型状态。
