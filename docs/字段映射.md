# 劳动者港湾智能助手字段映射

更新时间：2026-07-09

## 1. 说明

本文档用于连接三类内容：

- 前端 TypeScript 字段
- 数据库 SQL 草案字段
- 后续 API/后端落库字段

当前真实校验入口在 `src/data/schemaContracts.ts`，测试文件为 `src/data/schemaContracts.test.ts`。

## 2. Harbor 港湾表

| 前端字段 | 数据库字段 | 必填 | 说明 |
|---|---|---|---|
| id | id | 是 | 港湾 ID |
| name | name | 是 | 港湾名称 |
| district | district | 是 | 区县 |
| businessArea | business_area | 是 | 商圈 |
| address | address | 是 | 地址 |
| longitude | longitude | 是 | 经度 |
| latitude | latitude | 是 | 纬度 |
| walkingTimeMinutes | walking_time_minutes | 否 | 步行时间，真实地图接入后由地图服务返回 |
| distanceMeters | distance_meters | 否 | 距离，地图失败时可使用直线距离 |
| status | status | 是 | open/closed/temp_closed |
| statusReason | status_reason | 否 | 关闭或临时关闭原因 |
| openingHours | opening_hours | 是 | 开放时间 |
| crowdLevel | crowd_level | 是 | quiet/moderate/crowded/unknown |
| updatedAt | updated_at | 是 | 更新时间 |
| lastVerifiedAt | last_verified_at | 是 | 最近核验时间 |
| dataFreshnessStatus | data_freshness_status | 是 | fresh/stale |

## 3. Facility 设施表

| 前端字段 | 数据库字段 | 必填 | 说明 |
|---|---|---|---|
| id | id | 是 | 设施 ID |
| harborId | harbor_id | 是 | 所属港湾 |
| type | type | 是 | drinking_water/charging/rest 等 |
| name | name | 是 | 设施名称 |
| available | available | 是 | 是否可用 |
| status | status | 是 | normal/fault/maintenance/unknown |

## 4. Report 反馈表

| 前端字段 | 数据库字段 | 必填 | 说明 |
|---|---|---|---|
| reportId | id | 是 | 反馈 ID |
| harborId | harbor_id | 是 | 关联港湾 |
| category | category | 是 | 问题分类 |
| description | description | 是 | 文字描述 |
| imageUrl | image_url | 否 | 图片地址，P0-M 可为空 |
| imageUploadStatus | image_upload_status | 是 | not_provided/uploaded/failed |
| imageUploadNote | image_upload_note | 否 | 图片上传失败或降级说明 |
| status | status | 是 | pending/processing/resolved/rejected |
| createdAt | created_at | 是 | 创建时间 |

## 5. Work Order 工单表

| 前端字段 | 数据库字段 | 必填 | 说明 |
|---|---|---|---|
| workOrderId | id | 是 | 工单 ID |
| reportId | report_id | 是 | 关联反馈 |
| harborId | harbor_id | 是 | 关联港湾 |
| status | status | 是 | pending/processing/resolved/closed |
| assignee | assignee | 否 | 处理人 |
| resultNote | result_note | 否 | 处理结果 |
| createdAt | created_at | 是 | 创建时间 |
| updatedAt | updated_at | 是 | 更新时间 |

## 6. 当前校验策略

当前测试会检查：

- seed 数据字段与字段契约一致。
- SQL 草案包含核心表字段。
- 关键枚举值出现在 SQL CHECK 约束中。

后续接入真实后端时，可在此基础上继续增加：

- JSON Schema 校验。
- API 响应契约测试。
- 数据库迁移脚本校验。
- 端到端接口测试。
