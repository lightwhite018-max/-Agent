# 劳动者港湾智能助手迭代记录

更新时间：2026-07-09

## 当前版本

当前项目已推进到 `V0.7 接入准备原型`。这一阶段的目标不是上线完整公共服务平台，而是围绕 PRD 的 P0-M 验收标准，证明“劳动者能不能更快找到附近可用港湾，并把问题反馈形成工单”，同时把地图、天气、后端接入前的降级体验做成可演示状态。

当前核心链路：

```text
自然语言/快捷需求 -> 需求识别 -> 港湾推荐 -> 港湾详情 -> 文字反馈 -> 工单生成 -> 简易管理
```

## 版本演进

### V0.1 文档到可运行原型

完成内容：

- 初始化 React + TypeScript + Vite 项目。
- 建立 README、启动脚本和基础工程配置。
- 内置 HB001-HB004 样例港湾数据。
- 实现规则 Agent：
  - 喝水
  - 充电
  - 休息
  - 避雨
  - 如厕
  - 反馈
  - 模糊需求追问
- 实现基础推荐规则：
  - 开放状态过滤
  - 设施类型匹配
  - 设施状态过滤
  - 步行时间排序
  - 拥挤程度弱排序
- 实现港湾详情、文字反馈和工单生成。

验证结果：

- “我想喝水”可推荐 HB001。
- “下雨了还想充电”可识别充电和室内条件。
- “我想找个地方”会进入追问。
- 文字反馈可生成反馈编号和工单编号。

### V0.7 地图和天气接入准备

完成内容：

- 导航按钮从静态展示改为可点击交互。
- 地图未配置时展示结构化降级结果：
  - 确认目的地
  - 展示港湾地址
  - 提示直线距离仅供参考
- 天气状态条可随用户输入切换降雨、高温、低温场景。
- 降雨场景提示优先考虑室内避雨、充电和雨具。
- 高温场景提示优先考虑室内休息、饮水和降温避暑。
- 低温场景提示优先考虑室内休息、热水和短暂停留。
- 外部服务测试覆盖导航降级步骤和天气场景提示。
- 反馈面板支持模拟图片上传成功、失败和不上传三种状态。
- 图片上传失败时仍按文字反馈生成工单，并记录降级说明。
- 管理端工单列表支持将工单推进为待处理、处理中、已解决、已关闭。
- 推荐日志可从预览保存为本地记录，保留输入、意图、结果港湾、降级状态和位置来源。
- 首页演示状态栏改为开放港湾、当前港湾、待处理工单和推荐日志，便于讲解闭环进度。
- 管理端摘要区补充已解决工单数量。

## P0-M 验收状态

当前验收面板内置 15 条 PRD P0-M 用例。

状态分类：

- 已覆盖：原型中已可直接演示。
- 演示增强：原型已表达交互或降级策略，但真实第三方服务待接入。
- 待接入：需要地图、天气或后端能力才能完整实现。

当前重点状态：

| 范围 | 状态 | 说明 |
|---|---|---|
| 自然语言喝水推荐 | 已覆盖 | 可识别 drinking_water 并推荐 HB001 |
| 多需求推荐 | 已覆盖 | 可识别 charging + indoor |
| 模糊需求追问 | 已覆盖 | 提供快捷选项 |
| 定位拒绝降级 | 已覆盖 | 可手动选择区县/商圈 |
| 设施筛选 | 已覆盖 | 故障设施不计入满足条件 |
| 开放状态判断 | 已覆盖 | 关闭港湾不进入首选推荐 |
| 推荐排序 | 已覆盖 | 有可解释排序因子 |
| 港湾详情 | 已覆盖 | 展示状态、设施、更新时间 |
| 文字反馈 | 已覆盖 | 可生成反馈和工单 |
| 工单生成 | 已覆盖 | 管理端可查看并更新工单状态 |
| 地图路线 | 演示增强 | 当前可点击查看地址和直线距离降级结果，真实路线需要地图 API |
| 图片反馈 | 演示增强 | 可模拟图片上传成功或失败，失败时继续按文字反馈生成工单 |
